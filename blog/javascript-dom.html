The DOM

DOM is an acronym for Document Object Model.
It is a representation, in objects, of the entire HTML document. It is much like a family tree, but instead of family members, we are referring to 'nodes'.
Here is a chart showing an example of the DOM.


An example of interacting with the DOM would be when we access HTML elements using their tag name, class name or id with the querySelector or querySelectorAll method.
Below is an example of code showing how to select the first element with the class 'cheese-fridge':
let cheeseFridge = document.querySelector(".cheese-fridge")

Here is an example of how to access 'child' elements within the selected element:

let cheeseType = cheeseFridge.querySelector(".cheese-type");
cheeseType.textContent = "Current Cheese Type";

Accessing Data From Objects vs Arrays

An object is defined as a collection of key-value pairs, which are data structures used to represent a piece of information. The key is an identifier for the data, which tells you what it represents. The value is the data associated with the key.

Objects are best used when your data represents one thing with named properties and when you will be manipulating individual properties of the data rather than performing manipulations over the data as a whole entity, which would be best suited to an array.

Another way to tell if it is appropriate to use an array is if the order of the data matters or if it is a group of things of the same type.

The way that the data needs to be accessed (numerical index vs unique keys) is informed by considering the aforementioned situations, knowing that objects are unordered and arrays are ordered.

Functions and Why They Are Useful

A function is a reusable block of code created to perform a particular task, such as returning a string as shown in the code example below:

// function greet() {
return 'Hey there!' }
let salutation = greet() //

Functions are helpful as they let you break down complex tasks into more manageable steps, because they are like smaller programs within your main program. A function can be 'called', which means giving the function an instruction to run its defined set of instructions. In our example above, the last line of code is calling the function.

Once a function has been defined it can be called multiple times with different inputs inserted if needed. It is easier to debug and test code when functions are used, as each one undertakes a specific task.
